# 🏦 Bank Churn Prediction App

**MLOps Assignment 02** — A machine learning application that predicts bank customer churn using a trained classification model, served via a **FastAPI** backend and consumed through a **Streamlit** frontend.

## 🚀 Getting Started

### Prerequisites

- Python 3.10+
- pip

### 1. Clone the Repository

```bash
git clone https://github.com/Therm1te/Bank-Churn-Prediction-MLOps
cd Bank-Churn-Prediction-MLOps
```

### 2. Create Virtual Environment

```bash
python -m venv venv

# Windows
.\venv\Scripts\activate

# Linux/macOS
source venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install pandas scikit-learn xgboost matplotlib seaborn fastapi uvicorn streamlit requests numpy
```

### 4. Generate Dataset

```bash
python download_dataset.py
```

### 5. Train the Model

```bash
python notebooks/train_model.py
```

This will:

- Perform EDA and save plots to `notebooks/eda_plots/`
- Train Logistic Regression, Random Forest, and XGBoost models
- Evaluate and compare models (Accuracy, F1, ROC-AUC)
- Perform hyperparameter tuning on the best model
- Save the final model to `backend/model.pkl`

### 6. Run the Backend (FastAPI)

```bash
cd backend
uvicorn app.main:app --reload
```

The API will start at **<http://127.0.0.1:8000>**

- `GET /` → Welcome message
- `POST /predict` → Predict churn
- `GET /docs` → Interactive Swagger documentation

### 7. Run the Frontend (Streamlit)

Open a **new terminal**, activate the venv, and run:

```bash
cd frontend
streamlit run app.py
```

The UI will open at **<http://localhost:8501>**

## 🧠 Model Details

### Features Used

| Feature | Type | Description |
|---------|------|-------------|
| CreditScore | int | Customer's credit score (350-850) |
| Geography | str | Country: France, Germany, Spain |
| Gender | str | Male or Female |
| Age | int | Customer's age (18-92) |
| Tenure | int | Years with the bank (0-10) |
| Balance | float | Account balance |
| NumOfProducts | int | Number of bank products (1-4) |
| HasCrCard | int | Has credit card (0/1) |
| IsActiveMember | int | Is active member (0/1) |
| EstimatedSalary | float | Estimated annual salary |

### Preprocessing

- **Dropped columns:** RowNumber, CustomerId, Surname (non-predictive)
- **Categorical encoding:** OneHotEncoding for Geography and Gender (`drop_first=True`)
- **Scaling:** StandardScaler applied for Logistic Regression

### Models Trained

1. **Logistic Regression** — Simple baseline
2. **Random Forest** — Ensemble method
3. **XGBoost** — Gradient boosting

### Evaluation Metrics

- Accuracy
- F1 Score
- ROC-AUC
- Confusion Matrix
- Classification Report (Precision, Recall)

The best model is automatically selected and saved.

## 🏗️ Architecture

```
┌─────────────────┐        ┌──────────────────┐        ┌──────────────┐
│   Streamlit UI  │──POST──│   FastAPI API     │──load──│  model.pkl   │
│  (Frontend)     │  /predict│  (Backend)       │        │ (ML Model)   │
│  localhost:8501 │◄─JSON──│  localhost:8000   │        └──────────────┘
└─────────────────┘        └──────────────────┘
```

- **Frontend** is decoupled from the backend
- **API** acts as middleware between the UI and ML model
- **Model** can be updated independently without changing the UI
- Multiple clients (web, mobile, desktop) can call the same API

## 📋 Assumptions

- The dataset uses the standard Bank Churn Modelling schema
- France is used as the baseline for Geography encoding
- Female is used as the baseline for Gender encoding
- The model is retrained from scratch each time `train_model.py` runs
